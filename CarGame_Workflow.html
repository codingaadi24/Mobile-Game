High-Graphics Car Game

Development Workflow & Timeline (Unity)

1

Preparation & Setup 1 - 2 Weeks

  • Install Unity Hub, Unity Editor (LTS), Android SDK, NDK, and OpenJDK.
  • Install and configure Blender for 3D modeling.
  • Create a new Mobile-Optimized Universal Render Pipeline (URP) project.
  • Model the 3D car and design the game map/environment in Blender.
  • Source audio and additional assets.
2

Core Driving Mechanics 2 - 4 Weeks

  • Export the car from Blender, then import and rig the 3D model.
  • Configure Unity Wheel Colliders or a Car Physics package.
  • Script Mobile UI controls (steering wheel, pedals) using C#.
  • Implement an dynamic follow camera system.
3

Level & Environment Design 3 - 6 Weeks

  • Import the custom Blender map and verify scale/collisions.
  • Refine layout and use Spline tools for smooth roads if needed.
  • Populate the world with modular buildings, trees, and barriers.
  • Bake mobile-friendly lighting and optimize with the Universal Render Pipeline (URP).
4

Game Loop & Systems 3 - 5 Weeks

  • Build lap tracking and invisible checkpoints.
  • Create Main Menu, UI overlays, and pause screens.
  • Implement basic Enemy AI to follow splines.
5

Polish, Audio & Graphics 2 - 4 Weeks

  • Add engine sounds, tire squeals, and collision SFX.
  • Implement VFX (tire smoke, dust, sparks).
  • Fine-tune car paint materials and post-processing.
6

Optimization & Publishing 2 - 4 Weeks

  • Compress textures and lower polygon counts for mobile stability.
  • Package and test the `.apk` on a physical Android device.
  • Prepare store assets (icon, screenshots) for Google Play.
Total Estimated Timeline: 3 to 6 Months
(Based on 15–20 hours per week as a solo dev)
